# SS2D-ViT v1 결과 분석: 왜 복원 이미지가 이렇게 blurry한가

## 1. 문제 현상

200 epoch 학습 후 **BEST** 샘플 (Rank 1 / 4492) 결과:

| 지표 | 값 | U-Net 기대값 |
|------|-----|-------------|
| SSIM | 0.7043 | 0.73 ~ 0.78 |
| PSNR | 35.68 dB | 30 ~ 33 dB |
| NMSE | 0.0104 | 0.005 ~ 0.01 |
| L1 | 6.2436 | - |

PSNR은 양호해 보이지만, 실제 시각화를 보면:
- **복원 이미지가 aliased 입력보다 더 blurry** (구조 디테일이 모두 사라짐)
- Edge, 조직 경계, 미세 구조가 전부 뭉개짐
- 오차 맵(Error Map)에서 뇌 구조 경계부에 집중된 에러 확인

> PSNR이 높은 이유: blurry하지만 **평균적으로 밝기가 GT와 가깝기 때문**.
> PSNR은 pixel-wise MSE 기반이라, "안전한 평균값"을 출력하면 수치는 올라감.
> 실제 지각적 품질은 SSIM이 더 정확히 반영하며, 0.70은 U-Net 대비 현저히 낮음.

## 2. v1 아키텍처 데이터 흐름

```
입력: aliased image (B, 32, 320, 320)     k-space (B, 32, 320, 320)
          │                                       │
          ▼                                       ▼
    ┌─────────────┐                        ┌────────────┐
    │ 32×32 패치화 │                        │    SS2D    │
    │ 100 토큰     │                        │ d_inner=32 │
    │ (384 dim)    │                        │ d_state=8  │
    └──────┬──────┘                        └─────┬──────┘
           │                                      │
    ViT Encoder (6L)                         4방향 SSM
    ViT Decoder (6L)                              │
           │                                      │
    final_linear                                  │
    512 → 65,536                                  │
           │                                      │
    Rearrange + PixelShuffle                      │
    (B, 256, 320, 320)                    (B, 20, 320, 320)
           │                                      │
           └──────────────┬───────────────────────┘
                          │
              ┌───────────┴───────────┐
              │ cat: 256 + 32 + 20    │
              │      = 308 채널       │
              └───────────┬───────────┘
                          │
              ┌───────────┴───────────┐
              │ Conv2d(308, 1, 3×3)   │  ← 파라미터 2,773개
              └───────────┬───────────┘
                          │
              출력: (B, 1, 320, 320)
```

## 3. 원인 분석

### 원인 1: 32×32 패치가 공간 정보를 파괴 (가장 치명적)

```
v1: patch_size = (32, 32)
    320 / 32 = 10  →  10×10 = 100 패치

    각 패치: 32 × 32 × 32ch = 32,768 값
    → LayerNorm → Linear → 384 차원 토큰으로 압축

    압축비: 32,768 : 384 = 85 : 1
```

32×32 영역의 모든 공간 정보가 384개 숫자로 압축된다.
이는 32×32 픽셀 블록을 하나의 "요약"으로 만드는 것과 같다.

**MRI 재구성에서 SSIM/시각적 품질을 결정하는 것은 edge, texture, 조직 경계 같은 fine detail인데,
이 정보가 인코딩 단계에서 이미 비가역적으로 손실된다.**

비교:
- 이미지 복원용 ViT (SwinIR, Restormer): patch_size = 4~8
- U-Net: 3×3 conv로 픽셀 단위 처리, 공간 정보 100% 보존

### 원인 2: 정보 병목 (Information Bottleneck)

전체 정보량 추적:

```
입력 정보:  32ch × 320 × 320 = 3,276,800 값

인코더 병목: 100 토큰 × 384 dim = 38,400 값    ← 전체 정보의 1.2%만 통과
디코더 출력: 100 토큰 × 512 dim = 51,200 값

final_linear 확장: 100 × 65,536 = 6,553,600 값  ← "확장"이지만...
```

**핵심**: `final_linear`이 512 → 65,536으로 확장하지만, **이미 38,400 병목에서 잃어버린 정보는 복구 불가능**.
네트워크는 학습 데이터에서 본 패턴의 "평균"을 출력할 수밖에 없다 → **blurry output**.

### 원인 3: 최종 합성이 Conv2d 1개 (파라미터 2,773개)

```python
# v1 코드
num_ch_last = 256 + 32 + 20  # = 308 채널
self.last = nn.Conv2d(308, 1, kernel_size=3, padding=1, bias=True)
# 파라미터: 308 × 3 × 3 × 1 + 1 = 2,773개
```

308채널의 풍부한 특징을 단 하나의 3×3 conv로 1채널로 변환한다.
이것은 사실상 **308채널의 가중 평균**에 불과하다.

- ViT 브랜치가 구조 정보를 담고 있어도
- SS2D 브랜치가 k-space 보정 정보를 담고 있어도
- 이 두 정보를 **비선형적으로 결합하여 정제할 능력이 없다**

비교: U-Net은 수십 개의 conv 레이어로 점진적 특징 정제.

### 원인 4: SS2D 용량 부족 (d_inner=32)

```
SS2D 입력:  32채널
→ Linear(32, 32): 내부 차원 d_inner = 32
→ 4방향 SSM 각각 d_inner=32, d_state=8
→ merge: 4×32=128 → 32
→ 출력 투영: 32 → 20채널
```

- **d_inner=32**는 320×320 MRI 데이터의 복잡한 k-space 상관관계를 잡기에 극히 작다
- 참고: VMamba는 분류 작업에서도 d_inner=192+ 사용
- 입력 32ch를 32차원으로 투영 → **정보 보존이 아니라 그냥 차원 유지**, 표현력 향상 없음
- 4방향 SSM의 상태 차원 d_state=8도 매우 작아, 장거리 의존성 포착 불가

### 원인 5: SSIM Loss가 사실상 무시됨

```python
LAMBDA_SSIM_PER_PIXEL = 0.2
loss = loss_l1 + 0.2 * (1 - SSIM)
```

Epoch 200 기준 실제 기여도:

| 손실 항 | 값 | 비중 |
|---------|-----|------|
| L1 | ~10.3 | **99.3%** |
| 0.2 × (1-SSIM) | ~0.07 | **0.7%** |

SSIM이 전체 loss의 0.7%만 차지 → **구조적 유사도 최적화가 거의 진행되지 않음**.

L1 loss만으로 학습하면:
- 모든 가능한 출력의 pixel-wise 평균이 L1을 최소화
- 이 "평균"은 **blurry output** 그 자체
- SSIM이 실질적으로 기여해야 edge, texture가 보존됨

### 원인 6: Train-Val 과적합 갭

```
Epoch 200:
  Train SSIM: 0.665  (계속 상승 중)
  Val   SSIM: 0.599  (epoch ~100 이후 정체)
  Gap:        0.066
```

- 모델이 학습 데이터를 암기하고 있음 (train SSIM은 계속 올라감)
- 하지만 일반화하지 못함 (val SSIM은 epoch 100 이후 0.60 근처에서 정체)
- 이는 아키텍처의 표현력 한계가 아닌, **학습 가능한 패턴의 한계**를 보여줌
- 즉, 모델은 "32×32 블록 단위로 요약된 정보"에서 **복원할 수 있는 것의 한계에 도달**

## 4. 왜 복원 이미지가 aliased 입력보다 더 blurry한가

이것이 가장 이상하게 보이는 점. aliased 입력에는 사실 **모든 고주파 정보가 포함**되어 있다
(단지 겹침 아티팩트가 있을 뿐). 모델의 역할은 아티팩트를 제거하면서 구조를 보존하는 것.

하지만 v1 모델은:

1. **ViT 인코더가 32×32 패치로 압축** → 고주파 정보 파괴
2. **디코더가 저주파 근사만 복원** → 세밀한 구조 재생성 불가
3. **SS2D가 k-space 보정을 하지만 용량 부족** → 미미한 기여
4. **단일 Conv2d가 이 모든 것을 합산** → 가중 평균 = blurry
5. **L1 중심 학습이 "안전한 평균값"을 선호** → 선명한 예측은 risk가 높음

결과: **입력의 고주파 정보가 모델을 통과하면서 제거되어, 출력이 입력보다 더 흐릿해진다.**

## 5. v1 수치 요약

### 학습 추이 (200 epochs)

| 시점 | Train Loss | Train SSIM | Val SSIM | Val PSNR | Val NMSE |
|------|-----------|-----------|---------|---------|---------|
| Epoch 1 | 48.72 | 0.395 | 0.435 | 19.36 dB | 0.469 |
| Epoch 50 | 16.32 | 0.567 | 0.575 | 25.50 dB | 0.155 |
| Epoch 100 | 12.85 | 0.621 | 0.596 | 26.30 dB | 0.135 |
| Epoch 200 | 10.26 | 0.665 | 0.599 | 26.45 dB | 0.128 |

- **Epoch 50~200**: Train SSIM은 0.567→0.665로 계속 상승하지만, Val SSIM은 0.575→0.599로 거의 정체
- 100 epoch 이후 150 epoch을 더 학습해도 **Val 성능 개선이 거의 없음**

### v1 Config (수정 전)

```python
PATCH_SIZE = (32, 32)                        # 100 패치
LAMBDA_SSIM_PER_PIXEL = 0.2                  # SSIM 기여 0.7%
NUM_SS2D_D_INNER = 32                        # SSM 내부 차원
NUM_SS2D_D_STATE = 8                         # SSM 상태 차원
NUM_VIT_DECODER_FINAL_LINEAR_OUT_CH = 256    # 디코더 출력 채널
NUM_VIT_DECODER_FINAL_LINEAR_OUT_FEAT = 16   # 디코더 출력 해상도
# 최종 합성: nn.Conv2d(308, 1, 3, padding=1)  # 파라미터 2,773개
```

## 6. v2에서 적용한 개선 사항

| 항목 | v1 | v2 | 기대 효과 |
|------|-----|-----|----------|
| Patch size | 32×32 (100패치) | **16×16 (400패치)** | 인코딩 압축비 85:1 → 21:1, 공간 정보 4배 보존 |
| 최종 합성 | Conv2d 1개 (2,773 params) | **RefinementBlock** (3×ResBlock, ~120K params) | 비선형 특징 결합 + 점진적 정제 |
| SSIM weight | 0.2 (기여 0.7%) | **1.0** (기여 ~3.5%) | 구조 보존 학습 강화 |
| Decoder 출력 | 256ch, feat=16 | **64ch, feat=8** | 메모리 절약, 1회 upsample 유지 |
| (ETER) GRU hidden | 2 | **4** | GRU 표현력 2배 |

## 7. v2 해결 상태 체크리스트

| # | 문제 | 상태 | 비고 |
|---|------|------|------|
| 1 | Patch size 32×32 → 공간 정보 파괴 | **해결** | 16×16으로 축소, 패치 수 100→400 |
| 2 | Decoder 업샘플 bottleneck | **해결** | OUT_CH=256→64, FEAT=16→8 |
| 3 | 최종 합성 Conv2d 1개 (2,773 params) | **해결** | RefinementBlock (3×ResBlock, ~120K params) |
| 4 | SSIM loss 기여 0.7% | **해결** | weight 0.2→1.0 |
| 5 | Train-Val 과적합 갭 (0.066) | **부분 해결** | 구조 개선으로 표현력 병목 완화 → 과적합 줄어들 가능성 높음. 명시적 정규화(dropout, augmentation 등)는 미추가. v2 학습 후 gap 확인 필요 |
| 6 | Data Consistency layer 부재 | **미해결** | 현재 모델이 single-channel magnitude 출력이고 coil sensitivity map이 없어서 multi-coil k-space와 직접 비교하는 DC layer 삽입 불가. 적용하려면 모델 출력을 complex multi-coil로 변경하거나 sensitivity map estimation 모듈 추가 필요 (아키텍처 대폭 변경). v2 결과 확인 후 판단 |
| 7 | ETER GRU hidden=2 (표현력 극소) | **해결** | hidden=4, bidirectional 출력 4→8ch |
